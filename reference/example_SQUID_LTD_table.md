# Example SQUID LTD Table

Example dataset for SQUID long-term deployment analysis.

## Usage

``` r
example_SQUID_LTD_table
```

## Format

A data frame with 374 rows and 51 columns:

- ASSESSMENT_UNIT_ID:

  Assessment unit identifier

- ASSESSMENT_UNIT_NAME:

  Assessment unit name

- AQUATIC_LIFE:

  Aquatic life use category

- PROJECT_NAME:

  Project name

- STATION_ID:

  Station identifier

- STATION_NAME:

  Station name

- SAMPLING_EVENT_TYPE:

  Type of sampling event

- COMMENTS:

  General comments

- SE_STAFF:

  Sampling event staff

- DATA_LOGGER_NAME:

  Name of data logger

- DATA_LOGGER_TYPE:

  Type of data logger

- DATA_LOGGER_NO:

  Data logger serial number

- ACT_ID:

  Activity identifier

- ACTII_ID:

  Secondary activity identifier

- ACT_START_DATE:

  Activity start date

- ACT_END_DATE:

  Activity end date

- DAYS_DEPLOYED:

  Number of days deployed

- ACTIVITY_TYPE:

  Type of activity

- ACMED_NAME:

  Measurement name

- ACTIVITY_COMMENTS:

  Comments on activity

- CHARACTERISTIC_NAME:

  Characteristic measured

- MEASUREMENT:

  Measurement value

- UNITS:

  Units of measurement

- WQS_REFERENCE:

  Water quality standards reference

- `> 3 days deployed`:

  Indicator if deployed \> 3 days

- ASSESSABILITY_QUALIFIER_CODE:

  Assessability qualifier code

- DO_WQC:

  Dissolved oxygen criterion

- `DO Min < DO WQS?`:

  Flag if DO minimum \< WQS

- PH_LOW:

  Lower pH threshold

- `pH Min < pH Low?`:

  Flag if pH minimum \< low threshold

- PH_HIGH:

  Upper pH threshold

- `pH Max > pH High?`:

  Flag if pH maximum \> high threshold

- `pH Max > 9.5?`:

  Flag if pH maximum \> 9.5

- TEMP_WQC:

  Temperature criterion

- `Temp Max > Temp WQC?`:

  Flag if temperature exceeds WQC

- SC_WQC:

  Specific conductance criterion

- `Max SC > SC WQC?`:

  Flag if SC exceeds WQC

- `Turbidity Max > 7?`:

  Flag if turbidity exceeds 7 NTU

- Turbdity Thresholds:

  Turbidity thresholds applied

- Turbidity Threshold Exceeded?:

  Flag if turbidity threshold exceeded

- NMAC_NOTES:

  Notes from NMAC review

- RES_COMMENTS:

  Result comments

- CHR_UID:

  Characteristic unique ID

- SE_TYPE_UID:

  Sampling event type ID

- RES_UID:

  Result unique ID

- ACT_UID:

  Activity unique ID

- ACTYP_UID:

  Activity type unique ID

- SE_UID:

  Sampling event unique ID

- MLOC_UID:

  Monitoring location unique ID

- PRJ_UID:

  Project unique ID

- AU_UID:

  Assessment unit unique ID
